##澄清问题（指南）

只问写一个清晰的PRD所需的最关键的问题。关注初始提示含糊不清或缺少基本上下文的领域。可能需要澄清的共同领域：

***问题/目标：**如果不清楚-“此功能为用户解决了什么问题？”
***核心功能：**如果含糊不清，请回答“用户应该能够执行哪些关键操作？”
***范围/边界：**如果是宽泛的-“此功能*不应该*做哪些具体的事情？”
***成功标准：**如果未说明-“我们如何知道此功能何时成功实现？”

**重要提示：**优先考虑对PRD清晰度有重大影响的问题。

###格式要求

-**给所有问题编号**（1、2、3等）
-**将每个问题的选项列为A、B、C、D等**以便于参考
-使用户能够轻松地用“1A、2C、3B”等选项进行响应

###示例格式

```
1.此功能的主要目标是什么？
A.改善用户入职体验
B.提高用户留存率
C.减轻支助负担
D.产生额外收入

2.此功能的目标用户是谁？
A.仅限新用户
B.仅限现有用户
C.所有用户
D.仅限管理员用户

3.此功能的预期时间表是什么？
A.紧急（1-2周）
B.高优先级（3-4周）
C.标准（1-2个月）
D.今后的审议（3个月以上）
```

输出格式：
- 更新后的完整 PRD（Markdown）
- 然后换行，列出新问题（如有）

不要添加解释性文字。
