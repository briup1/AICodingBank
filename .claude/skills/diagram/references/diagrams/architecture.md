# 架构图规范（Architecture Diagram）

> 仅 HTML/CSS 模式。D2/Mermaid 的图论布局无法精确控制分层堆叠。

---

## 适用场景

- 微服务系统架构
- 技术栈分层
- 模块依赖概览
- **评审场景**：方案评审、对外路演的整体架构图（叠加 `review-grade.md` 评审级规范）

---

## 模式选择

| 模式 | 何时用 | 布局特征 |
|------|--------|---------|
| **基础分层** | 默认场景，技术栈/微服务分层 | 纯纵向堆叠，无连线 |
| **主层 + 侧栏支撑层** | 评审/路演场景，需要展示监控/日志/CI/CD 等横切能力 | 主区域分层 + 右侧支撑栏（4:1 宽度比） |
| **顶部接入条 + 主层 + 底部基础设施** | 完整产品架构图（评审级最常见） | 三段式：客户端 → 业务分层 → 基础设施 |

---

## 设计原则

架构图展示的是**层级关系**（哪些组件在哪一层），不是调用关系（调用链路用时序图）。因此：
- **不画连线**：层与层之间用虚线分隔 + 协议标注代替箭头
- **分层堆叠**：从上到下按层级排列
- **同层等宽**：同层组件用色块平铺，等高等间距

**评审级例外**：评审场景下可在层与层之间加**单向粗箭头**指示主流向，但不画具体调用关系（仍由时序图承担）。

## 布局结构

```
┌──────────────────────────────────────────┐
│  标题 + 副标题                             │
├──────────────────────────────────────────┤
│  L-1  │ 层标签     │ [组件] [组件] [组件]   │
│       │            │                       │
├ - - - - - 协议/接口标注 - - - - - - - - - ┤
│  L-2  │ 层标签     │ [组件] [组件] [组件]   │
│       │            │                       │
├ - - - - - 协议/接口标注 - - - - - - - - - ┤
│  L-3  │ 层标签     │ [组件] [组件]          │
└──────────────────────────────────────────┘
```

## 层级配色

使用设计规范的层级色（L-1 ~ L-6），从上到下：

| 典型层级 | 层级色 | 组件色块 |
|---------|--------|---------|
| 客户端 / 接入层 | L-1 (Blue) | `#3B82F6` 填充 + 白字 |
| 业务服务层 | L-2 (Emerald) | `#10B981` 填充 + 白字 |
| 中间件 | L-3 (Amber) | `#F59E0B` 填充 + 白字 |
| 数据层 | L-4 (Violet) | `#8B5CF6` 填充 + 白字 |
| 基础设施 | L-5 (Rose) | `#F43F5E` 填充 + 白字 |
| 外部系统 | L-6 (Cyan) | `#06B6D4` 填充 + 白字 |

## 组件样式

| 属性 | 值 |
|------|-----|
| 形状 | 圆角矩形 |
| 圆角 | `4px` |
| 填充 | 对应层级色主色 |
| 文字 | `12px Medium` 白色 |
| 间距 | 组件间 `8px` |
| 高度 | `32px` 统一 |
| padding | `0 14px` |

## 层分隔

| 属性 | 值 |
|------|-----|
| 分隔线 | `1px dashed #E2E8F0` |
| 协议标注 | 居中，`10px Regular`，颜色 `#94A3B8` |
| 示例 | `HTTPS / WebSocket`、`gRPC / REST` |

---

## 整体尺寸

- 宽度：`860-960px`
- 背景：白色，`border-radius: 12px`，微阴影
- 内边距：`28-32px`
- 层高：自适应（根据组件行数）

---

## 层数限制

- 不超过 7 层
- 每层组件不超过 8 个
- 超出时按业务域拆分为多张架构图

## 主层 + 侧栏支撑层模式（评审级）

```
┌──────────────────────────────────────────────────┐
│  标题 + 副标题 + 顶部标签条                        │
├──────────────────────────────────────────────────┤
│ ┌──────────────────────────┐ ┌────────────────┐  │
│ │  L-1 客户端层             │ │   支撑能力     │  │
│ │ [Web] [App] [小程序]     │ │ ┌────────────┐ │  │
│ ├──────────────────────────┤ │ │ 监控告警   │ │  │
│ │  L-2 业务服务层           │ │ ├────────────┤ │  │
│ │ [服务A] [服务B] [服务C]   │ │ │ 日志中心   │ │  │
│ ├──────────────────────────┤ │ ├────────────┤ │  │
│ │  L-3 中间件层             │ │ │ 配置中心   │ │  │
│ │ [MQ] [Cache] [Search]    │ │ ├────────────┤ │  │
│ ├──────────────────────────┤ │ │ CI/CD      │ │  │
│ │  L-4 数据层               │ │ └────────────┘ │  │
│ │ [PG] [Redis] [Kafka]     │ │                │  │
│ └──────────────────────────┘ └────────────────┘  │
├──────────────────────────────────────────────────┤
│  底部图例                                         │
└──────────────────────────────────────────────────┘
```

### 侧栏布局参数

| 项目 | 值 |
|------|-----|
| 主区域 : 侧栏宽度 | 4 : 1（如 1200 : 280） |
| 侧栏 - 主区域间距 | 32px |
| 侧栏内每模块高度 | 64-80px |
| 侧栏标题 | 顶部居中，14px Bold，颜色 N-7 `#1E293B` |
| 侧栏背景 | N-2 `#F1F5F9`，圆角 12px |
| 侧栏模块文字色 | N-6 `#64748B`（避免抢主区域焦点） |

### 侧栏 vs 顶部/底部条

| 横切能力位置 | 适合内容 |
|-------------|---------|
| **右侧栏**（推荐） | 运维向：监控/日志/告警/CI/CD（"系统怎么跑起来"） |
| **底部条** | 数据基础设施：DB/Cache/MQ/CDN（"系统建立在什么之上"） |
| **顶部条** | 接入入口：Web/App/小程序/API（"用户从哪里进来"） |

完整产品架构图通常是 **顶部条 + 主分层 + 底部条 + 右侧栏** 的四向布局，最大信息密度。

## 禁忌

- 不画具体连线（用时序图/数据流图展示调用关系）
- 不使用渐变色
- 默认不使用 icon/emoji（**评审级模式例外**：参见 `review-grade.md`，允许 emoji 风格统一使用）
- 不使用设计规范以外的颜色
- **评审级**：标题区不能省，标签条 ≤ 6 个，连线类型 ≤ 3 种
